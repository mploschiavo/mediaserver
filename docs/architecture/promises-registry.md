# Promises Registry

`.ratchets/promises/promises.yaml` is the source of truth for "what works out-of-the-box after a fresh install." Every claim like *"after fresh install, Bazarr has English subtitles enabled"* lives there as one declarative entry.

For the human-readable index of every shipped promise, see [reference/promises.md](../reference/promises.md) (regenerate with `media-stack-render-promises`).

## Why this exists

The session that produced this registry had a recurring failure mode: a working feature got wired up via a one-shot manual mutation (PUT this, POST that). The mutation persisted on disk, the feature kept working through restarts, but `compose down -v` wiped the state and nothing re-applied it on the next `up`. Tests passed because they verified source-code shape, not live runtime state.

The promises registry forces three things to stay in sync:

1. **An `ensure_*` adapter** that re-asserts the desired state on every bootstrap run.
2. **A contract job** in `contracts/services/*.yaml` pointing at the adapter.
3. **A promise entry** in `.ratchets/promises/promises.yaml` with a probe that verifies the adapter actually worked.

If any of the three is missing, the meta-ratchet flags it.

## How the three layers consume the YAML

### 1. The meta-ratchet (`tests/unit/test_promises_registry.py`)

A unit test that runs in CI on every PR. It enforces:

- Every promise's `ensured_by` resolves to a real contract job (or `post_setup_handler`) somewhere in `contracts/services/*.yaml`.
- Every contract-job handler is importable.
- Every `ensure-*` job in contracts is referenced by at least one promise (no orphan adapters that quietly stop being needed).
- Every promise has the required fields and a valid probe type.
- Promise IDs are unique.

Failure means the registry has drifted from the rest of the codebase. The test won't pass until you either fix the broken reference or drop the orphan job.

### 2. The fresh-install verifier (`bin/verify-fresh-install.sh`)

Runs `compose down -v && wipe state && compose up`, waits for bootstrap to complete, then executes each promise's probe against the live stack. Fails on any miss.

This is the test that catches "the bootstrap config wiped by `compose down -v`, never re-applied" pattern. If a promise probe fails, either the adapter doesn't run, or it runs but doesn't actually achieve the state the probe expects — both worth knowing.

### 3. The reference page (`docs/reference/promises.md`)

Generated by `media-stack-render-promises` from the same YAML. Operators get a human-readable index of every guarantee the stack ships with.

## Probe types

Every promise has a `probe:` block:

| Type | Fields | Eval scope |
|---|---|---|
| `http_json` | `service`, `path`, `auth`, `assert` | `response` = parsed JSON |
| `http_text` | `service`, `path`, `auth`, `assert` | `data` = response body |
| `http_status` | `service`, `path`, `auth`, `assert` | `status`, `headers` |
| `file_json` | `path` (under `config_root`), `assert` | `data` = parsed JSON |
| `file_text` | `path`, `assert` | `data` = file text |

The `assert` field is a Python expression evaluated in the scope shown above. If it returns truthy, the promise holds.

### Auth

| Value | Behavior |
|---|---|
| `none` | No auth header. |
| `api_key` | Uses the discovered service key (Sonarr / Radarr / Bazarr / Prowlarr). |
| `jellyfin_key` | Uses the discovered Jellyfin API key. |
| `qbit_basic` | qBittorrent WebUI basic auth from `STACK_ADMIN_*`. |
| `controller_basic` | Controller's own basic auth. |

### Probe expression scope

Generator expressions in `any()` / `all()` use their own scope — names need to be in globals, not locals. The probe runner in `media-stack-probe-promises` already handles this; just write expressions naturally.

## Adding a new promise

1. **Write the `ensure_*` adapter** that re-asserts the desired state. It must be idempotent — every run leaves the system in the same end state.
2. **Register a contract job** pointing at the adapter (`phase: post`):
   ```yaml
   plugin:
     jobs:
       ensure-my-feature:
         handler: media_stack.services.apps.<service>.adapters:ensure_my_feature
         label: "Configure my feature"
         phase: post
         priority: 95
         requires: []
   ```
3. **Add a promise** to `.ratchets/promises/promises.yaml`:
   ```yaml
   - id: my-feature-works
     description: My feature is configured after a fresh install
     ensured_by: ensure-my-feature
     probe:
       type: http_json
       service: my-service
       path: /api/my-endpoint
       auth: api_key
       assert: "response.get('myFeature') == 'expected-value'"
   ```
4. **Re-render the reference** so docs reflect the new promise:
   ```bash
   media-stack-render-promises
   ```
5. **Run the meta-ratchet** locally:
   ```bash
   python3 -m pytest tests/unit/test_promises_registry.py -v
   ```

If steps 1+2 happen but step 3 doesn't, the meta-ratchet warns (orphan adapter); if step 3 happens without 1+2, the meta-ratchet fails hard (broken promise).

## Special case: `post_setup_handler`

Some service contracts use `plugin.post_setup_handler` instead of `plugin.jobs.<name>` (e.g. `unpackerr.yaml`). The post-setup handler runs through the same bootstrap pipeline as a job, just via the `container_post_setup_handlers` list.

The meta-ratchet treats `post_setup_handler` entries as valid `ensured_by` targets — synthetic name `<service>-post`. So a promise can reference `unpackerr-post` and the registry stays consistent.

## When a promise breaks

A failing promise during fresh-install verification points at one of three things:

1. **Adapter didn't run** — usually a phase-ordering bug or a missing prereq. Check the controller logs for the job's phase.
2. **Adapter ran but didn't achieve the state** — the API call succeeded but the value didn't stick. Common cause: schema migration in the upstream app changed a field name. Fix the adapter.
3. **Probe is wrong** — the assertion uses the old shape of the API response. Fix the probe.

Fix the root cause, then re-run the verifier. Don't relax the probe to make it pass — that defeats the point.
