# App UI Screenshots

Generated by:

```bash
bash bin/test/run-playwright-screenshots.sh <NODE_IP> [NAMESPACE]
```

For k8s deployments served via Envoy + Authelia, also export
`STACK_CONTROLLER_HOST`, `STACK_CONTROLLER_PREFIX`, and
`STACK_AUTHELIA_HOST` so the capture flow logs into Authelia
once and reuses the cookie. See
[../README.md](../README.md) for the full invocation.

Files are one-per-service and overwritten on each capture run.
Capture attempts authenticated login per app before taking the
screenshot. Filenames are derived from the service slug only
(``bazarr.png``, not ``bazarr.iomio.io.png`` / ``bazarr_local.png``)
so docs can reference ``apps/<service>.png`` without per-deployment
hostname noise.
